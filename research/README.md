# Research Notes

Research files preserve the evidence behind product and implementation choices. They are not marketing copy.

## Rules

- Label claims as **confirmed**, **reported**, or **assumption**.
- Include a source URL or internal reference and retrieval date for confirmed external claims.
- Do not represent unverified product APIs or access as available.
- Update the relevant note before implementation if an API choice depends on it.

## Files

- [`realtimekit-agent-dx.md`](realtimekit-agent-dx.md): problem evidence from RealtimeKit proposals and customer discussion.
- [`platform-capabilities.md`](platform-capabilities.md): capabilities that need current API validation before implementation.
- [`competitive-landscape.md`](competitive-landscape.md): adjacent products and differentiation.
- [`trial-methodology.md`](trial-methodology.md): evaluation validity and evidence rules.
